# 量化交易平台 — 功能需求規格書

> **版本**: v1.1
> **日期**: 2026-04-06
> **技術棧**: React 18 + TypeScript (前端) / Node.js + TypeScript (後端)
> **使用者**: 個人使用（單一使用者，無需帳號系統）
> **部署**: Google Cloud Run — Project ID: `stock-decision-assistant`
> **預算**: ≤ NT$1,000/月
> **市場**: 台股 + 美股（可切換）

---

## 1. 系統概覽

本平台為個人量化交易工具，整合四大核心模組於單一網頁應用中：

1. **交易儀表板 (Dashboard)** — 即時監控策略績效、持倉與損益
2. **回測平台 (Backtesting)** — 對歷史數據執行策略回測並分析結果
3. **策略管理系統 (Strategy Manager)** — 管理、部署、啟停交易策略
4. **數據分析工具 (Data Analytics)** — 市場數據的視覺化分析、篩選與探索

### 1.1 全域設計原則

- **單頁應用 (SPA)**：使用 React Router 實現頁面切換，無需重新載入
- **深色主題優先**：量化交易常需長時間盯盤，深色主題減輕視覺疲勞，同時提供淺色主題切換
- **響應式但桌面優先**：主要在大螢幕使用，但基本功能在平板上可用
- **即時更新**：透過 WebSocket 實現即時數據推送
- **模組化架構**：每個模組可獨立運作，透過共享的數據層通訊

### 1.2 導覽結構

- 左側固定側邊欄（可收合），包含四大模組入口
- 頂部導覽列顯示：系統狀態指示燈、全域搜尋、通知中心、主題切換、設定齒輪
- 每個模組內部使用 Tab 或子導覽切換子頁面

## 2. 模組一：交易儀表板 (Dashboard)

### 2.1 目的

提供策略運行狀態的即時全貌，讓使用者在一個畫面內掌握所有關鍵指標。

### 2.2 功能需求

#### 2.2.1 總覽卡片區

頁面頂部橫排顯示核心指標卡片：

| 卡片名稱 | 顯示內容 | 更新頻率 |
|---------|---------|---------|
| 總資產淨值 | 當前總淨值 + 日變動金額 + 日變動百分比 | 即時 |
| 今日損益 | 已實現 + 未實現損益，分別標示 | 即時 |
| 持倉數量 | 當前持有的部位總數 | 即時 |
| 運行中策略 | 正在執行的策略數 / 策略總數 | 即時 |
| 今日交易次數 | 買入次數 + 賣出次數 | 即時 |
| 勝率 | 今日已平倉交易的勝率 | 每筆平倉後 |

每張卡片需包含：主數值（大字）、副資訊（小字）、漲跌顏色標示（綠漲紅跌）、點擊可展開詳情。

#### 2.2.2 權益曲線圖

- **圖表類型**：面積圖 + 折線圖疊加
- **時間範圍選擇器**：今日、本週、本月、本季、本年、自訂區間
- **顯示項目**：
  - 總淨值曲線（主線）
  - 基準指數曲線（可疊加，例如 S&P 500、台灣加權指數）
  - Drawdown 區域（以紅色陰影標示回撤區間）
- **互動行為**：
  - 滑鼠懸停顯示 tooltip（日期、淨值、日報酬率、回撤幅度）
  - 可拖曳選取區間放大
  - 雙擊恢復全域視圖
- **右側統計面板**：
  - 總報酬率、年化報酬率、最大回撤、Sharpe Ratio、Sortino Ratio、Calmar Ratio

#### 2.2.3 持倉列表

以表格形式顯示所有當前持倉：

| 欄位 | 說明 |
|------|------|
| 標的代碼 | 股票/期貨/加密貨幣代碼 |
| 標的名稱 | 完整名稱 |
| 方向 | 多/空，以顏色區分 |
| 持倉數量 | 當前持有數量 |
| 平均成本 | 加權平均進場價格 |
| 當前價格 | 即時市價 |
| 未實現損益 | 金額 + 百分比 |
| 佔比 | 該部位佔總資產的百分比 |
| 所屬策略 | 產生此部位的策略名稱 |
| 操作 | 手動平倉、加倉、設定停損停利 |

**表格行為**：
- 點擊欄位標題排序（支援多欄排序）
- 欄位寬度可拖曳調整
- 點擊某行展開詳情（進場時間、進場原因、歷史加減倉記錄）
- 搜尋/篩選：依標的、策略、方向篩選
- 即時閃爍效果：價格變動時該格短暫閃綠（漲）或閃紅（跌）

#### 2.2.4 近期交易記錄

顯示最近 50 筆已完成交易：

| 欄位 | 說明 |
|------|------|
| 時間 | 成交時間 |
| 標的 | 代碼 + 名稱 |
| 方向 | 買入/賣出 |
| 數量 | 成交數量 |
| 價格 | 成交價格 |
| 損益 | 若為平倉交易，顯示該筆損益 |
| 策略 | 來源策略 |

**行為**：捲動載入更多、可匯出 CSV。

#### 2.2.5 策略績效摘要

以卡片網格或小型表格呈現每個策略的快速摘要：

- 策略名稱 + 狀態標籤（運行中 / 暫停 / 停止）
- 今日損益
- 總損益
- 勝率
- 持倉數
- 迷你權益曲線（Sparkline）
- 點擊跳轉至策略詳情頁

#### 2.2.6 告警與通知面板

右側或底部可收合面板，顯示系統告警：

- **告警等級**：緊急（紅）、警告（黃）、資訊（藍）
- **告警類型**：
  - 策略異常停止
  - 回撤超過閾值
  - API 連線中斷
  - 委託未成交超過設定時間
  - 自訂規則觸發
- 每條告警顯示：時間、等級圖示、訊息內容、相關策略、「已讀」按鈕
- 未讀數量在頂部導覽列以紅色徽章顯示

## 3. 模組二：回測平台 (Backtesting)

### 3.1 目的

讓使用者能在歷史數據上測試交易策略，評估績效指標，並與其他策略或基準進行比較。

### 3.2 功能需求

#### 3.2.1 回測設定頁面

**策略選擇區**：
- 下拉選單選擇已儲存的策略
- 或直接在內嵌程式碼編輯器中撰寫/修改策略邏輯
- 程式碼編輯器需支援：語法高亮（JavaScript/Python）、自動完成、錯誤提示、行號顯示

**參數設定區**：

| 參數 | 說明 | 輸入方式 |
|------|------|---------|
| 回測區間 | 開始日期 ~ 結束日期 | 日期選擇器 |
| 初始資金 | 回測起始資金 | 數字輸入 |
| 手續費模型 | 固定/百分比/階梯式 | 下拉 + 數字 |
| 滑價模型 | 固定點數/百分比/無 | 下拉 + 數字 |
| 標的池 | 可選多個標的 | 多選搜尋框 |
| 數據頻率 | Tick / 1分 / 5分 / 15分 / 1小時 / 日線 | 下拉 |
| 基準指數 | 用於比較的基準 | 下拉搜尋 |

**進階參數（可收合區塊）**：
- 最大持倉比例限制
- 單筆最大虧損限制
- 最大同時持倉數
- 是否允許放空
- 保證金比例（期貨/融資）

#### 3.2.2 回測執行

- 點擊「開始回測」按鈕後：
  - 顯示進度條（含預估剩餘時間）
  - 即時顯示正在處理的日期
  - 可隨時取消
- 回測完成後自動跳轉至結果頁面
- 支援同時排隊多個回測任務

#### 3.2.3 回測結果頁面

**績效摘要卡片**（與儀表板類似風格）：

| 指標 | 說明 |
|------|------|
| 總報酬率 | 起始至結束的總報酬 |
| 年化報酬率 | CAGR |
| 最大回撤 | 最大峰谷回撤百分比 + 持續天數 |
| Sharpe Ratio | 風險調整後報酬 |
| Sortino Ratio | 下行風險調整報酬 |
| Calmar Ratio | 年化報酬 / 最大回撤 |
| 勝率 | 獲利交易佔比 |
| 盈虧比 | 平均獲利 / 平均虧損 |
| 交易次數 | 總交易筆數 |
| 平均持倉天數 | 每筆交易的平均持有時間 |
| 最大連續虧損次數 | |
| 最大連續獲利次數 | |

**權益曲線圖**：
- 策略淨值曲線 vs 基準曲線
- Drawdown 圖（獨立子圖）
- 月報酬率熱力圖（類似 GitHub contribution chart）
- 年度報酬率長條圖

**交易明細表**：
- 與儀表板的交易記錄類似，但包含回測期間的所有交易
- 支援排序、篩選、匯出
- 點擊交易可在 K 線圖上標示進出場點

**K 線圖回放**：
- 顯示回測標的的 K 線圖
- 在圖上以箭頭標示買入（綠色向上箭頭）和賣出（紅色向下箭頭）點位
- 可逐步播放回測過程（快轉/慢放/暫停）
- 疊加技術指標（使用者可選擇）

#### 3.2.4 回測比較

- 選擇多個已完成的回測結果進行並排比較
- 比較表格：所有績效指標並排顯示，最佳值標示高亮
- 疊加權益曲線圖

#### 3.2.5 回測歷史

- 列表顯示所有歷史回測記錄
- 每條記錄顯示：策略名稱、回測區間、主要指標、執行時間、狀態
- 可刪除、重新執行、查看詳情
- 支援搜尋和篩選

## 4. 模組三：策略管理系統 (Strategy Manager)

### 4.1 目的

集中管理所有交易策略的生命週期：建立、編輯、測試、部署、監控、停止。

### 4.2 功能需求

#### 4.2.1 策略列表頁

以卡片或表格形式顯示所有策略：

| 欄位 | 說明 |
|------|------|
| 策略名稱 | 可點擊進入詳情 |
| 狀態 | 草稿 / 回測中 / 待部署 / 運行中 / 暫停 / 已停止 / 錯誤 |
| 類型標籤 | 趨勢追蹤 / 均值回歸 / 套利 / 高頻 / 自訂 |
| 適用標的 | 該策略交易的標的類型 |
| 建立時間 | |
| 最後修改時間 | |
| 即時損益 | 若運行中，顯示當前損益 |
| 操作按鈕 | 編輯 / 啟動 / 暫停 / 停止 / 複製 / 刪除 |

**頁面行為**：
- 狀態篩選 Tab（全部 / 運行中 / 暫停 / 草稿）
- 搜尋框
- 「新增策略」按鈕（右上角）
- 拖曳排序或優先級排序

#### 4.2.2 策略編輯器

**基本資訊區**：
- 策略名稱（文字輸入）
- 策略描述（多行文字）
- 類型標籤（下拉多選）
- 適用標的（多選搜尋框）

**程式碼編輯區**：
- 全功能程式碼編輯器（基於 Monaco Editor / CodeMirror）
- 支援語法高亮、自動完成、括號匹配、程式碼摺疊
- 內建策略模板可選用
- 左側檔案樹（若策略包含多個檔案）
- 底部終端面板（顯示執行日誌）

**策略模板**：
提供以下內建模板供快速開始：
- 移動平均線交叉策略
- RSI 超買超賣策略
- 布林通道突破策略
- 動量策略
- 配對交易策略
- 空白模板

**參數定義區**：
- 使用者可定義策略的可調參數
- 每個參數包含：名稱、類型（數字/文字/布林/選項）、預設值、最小值、最大值、步進值、說明
- 這些參數在回測和實盤時可以調整，不需改程式碼

#### 4.2.3 策略版本控制

- 每次儲存自動建立版本
- 版本列表：版本號、儲存時間、變更摘要
- 可比較兩個版本的差異（Diff View）
- 可回滾到任意歷史版本
- 可為版本加註標籤（例如 "v1.0-stable"、"實驗性"）

#### 4.2.4 策略部署

**部署設定**：
- 選擇交易帳戶 / 交易所 API
- 設定實際交易參數（可能與回測參數不同）
- 設定風控規則：
  - 單日最大虧損額度（觸發自動暫停）
  - 最大持倉比例
  - 單筆最大下單量
  - 最大回撤閾值
- 設定通知規則（何時發送告警）

**部署流程**：
1. 確認設定 → 2. 乾運行（Dry Run，模擬下單但不實際執行）→ 3. 正式啟動
- 乾運行結果需確認無誤後才能進入正式啟動

#### 4.2.5 策略監控

進入某個運行中策略的詳情頁後：

- **即時狀態**：運行時長、當前持倉、今日交易次數、今日損益
- **即時日誌**：策略產生的 log 訊息，支援等級篩選（DEBUG/INFO/WARN/ERROR）
- **即時 K 線**：顯示策略正在交易的標的 K 線，標示進出場點
- **歷史績效**：自部署以來的權益曲線和績效指標
- **快速操作**：暫停、恢復、緊急停止（一鍵平倉所有持倉）

## 5. 模組四：數據分析工具 (Data Analytics)

### 5.1 目的

提供市場數據的視覺化分析、篩選與探索工具，輔助策略開發和市場研究。

### 5.2 功能需求

#### 5.2.1 數據源管理

**支援的數據源**：
- 本地 CSV/JSON 檔案上傳
- API 串接（例如：Yahoo Finance、Binance、Alpha Vantage、台灣證交所）
- 資料庫連接（MongoDB / PostgreSQL）

**數據源設定頁**：
- 新增 / 編輯 / 刪除數據源
- 測試連線按鈕
- 數據更新排程設定（每分鐘 / 每小時 / 每日 / 手動）
- 數據使用量統計

#### 5.2.2 市場總覽

- **指數看板**：主要指數即時報價（可自訂顯示哪些指數）
- **熱力圖**：類似 Finviz 的板塊熱力圖，以顏色深淺表示漲跌幅度
- **漲跌排行**：今日漲幅 / 跌幅前 N 名
- **成交量排行**：異常放量標的
- **市場寬度指標**：上漲家數 / 下跌家數比率

#### 5.2.3 個股/標的分析

選擇單一標的後進入分析頁面：

**K 線圖區**：
- 支援：日K / 週K / 月K / 分鐘K
- 可疊加技術指標：
  - 趨勢類：MA、EMA、BOLL、MACD
  - 震盪類：RSI、KDJ、Stochastic
  - 成交量類：OBV、Volume MA
  - 其他：ATR、VWAP、Ichimoku Cloud
- 繪圖工具：趨勢線、水平線、斐波那契、矩形區域、文字標註
- 指標參數可即時調整
- 多圖並排比較（最多 4 個標的）

**基本面數據**（若可取得）：
- 財報摘要：營收、EPS、本益比、殖利率
- 以時間序列圖表呈現

**統計分析區**：
- 報酬率分布直方圖
- 波動率分析（歷史波動率、隱含波動率）
- 相關性矩陣（與其他標的的相關性）
- 季節性分析（月份平均報酬率）

#### 5.2.4 篩選器 (Screener)

自訂條件篩選標的：

**篩選條件**（可組合多條）：
- 價格範圍
- 成交量範圍
- 技術指標條件（例如：RSI < 30、MACD 黃金交叉）
- 漲跌幅範圍
- 市值範圍
- 自訂公式

**篩選器行為**：
- 條件之間支援 AND / OR 邏輯
- 可儲存篩選條件為「我的篩選器」
- 結果即時更新
- 結果以表格呈現，可排序
- 可將篩選結果匯出或直接加入策略的標的池

#### 5.2.5 自訂看板

- 使用者可建立多個自訂看板
- 每個看板使用網格佈局（Grid Layout），可自由配置 Widget
- **可用 Widget 類型**：
  - K 線圖（迷你版）
  - 即時報價表
  - 自選股列表
  - 技術指標圖
  - 熱力圖
  - 新聞 Feed
  - 自訂文字筆記
  - 權益曲線圖
- Widget 可拖曳調整位置和大小
- 看板可重新命名、複製、刪除
- 自動儲存佈局

#### 5.2.6 數據匯出

- 支援格式：CSV、JSON、Excel
- 可選擇匯出的數據範圍和欄位
- 批次匯出功能

## 6. 共用功能

### 6.1 全域搜尋

- 快捷鍵 `Ctrl+K` 或 `/` 開啟搜尋面板
- 搜尋範圍：策略名稱、標的代碼/名稱、告警內容、回測記錄
- 搜尋結果分類顯示，點擊直接跳轉

### 6.2 設定頁面

- **一般設定**：語言（繁中/英文）、時區、日期格式、數字格式
- **交易所 API 管理**：新增/編輯 API Key，測試連線，選擇預設交易所
- **通知設定**：告警通知方式（網頁推播 / Email / LINE / Telegram）
- **顯示設定**：主題（深色/淺色/自動）、預設圖表顏色、字體大小
- **數據設定**：數據更新頻率、快取策略、本地儲存空間管理
- **匯入匯出**：匯出所有設定/策略/數據為備份檔，從備份檔還原

### 6.3 鍵盤快捷鍵

| 快捷鍵 | 功能 |

### 6.4 錯誤處理

- WebSocket 斷線自動重連（指數退避策略）
- 關鍵操作（平倉、啟動策略）需二次確認

## 7. 非功能需求

### 7.1 效能

- 首頁載入時間 < 3 秒
- 即時數據更新延遲 < 500ms
- K 線圖渲染 10,000+ 根 K 線流暢（使用 Canvas/WebGL）
- 回測引擎支援 10 年日線數據在 30 秒內完成

### 7.2 數據持久化

- 策略程式碼和設定存儲於 Cloud Firestore（免費額度足夠個人使用）
- 歷史行情數據存於 Cloud Storage（JSON/CSV 格式，按標的/日期分目錄）
- 回測結果存於 Firestore，可選擇保留或刪除
- 用戶設定存於 Firestore `settings` collection
- 定期自動備份至 Cloud Storage

### 7.3 安全性

- API Key 存於 Google Cloud Secret Manager，前端不可讀取明文
- 部署於 Cloud Run，透過 IAM 控制存取
- 前端透過 Cloud Run 提供，無需額外暴露 port
- 若需額外保護，可設定 Cloud Run 需要認證（IAP）

### 10.4 市場切換機制

```
UI 左上方市場切換器:
┌──────────────────────┐
│  [🇹🇼 台股] [🇺🇸 美股]  │
└──────────────────────┘
```

- 切換市場後，所有模組的數據來源、標的池、篩選條件自動對應
- 各市場獨立的設定（交易時間、手續費率、稅率等）
- 台股特有：除權息資料、三大法人買賣超
- 美股特有：盤前盤後數據、選擇權鏈

### 10.5 數據更新流程

```
Cloud Scheduler (每日排程)
   │
   ├─ 06:00 UTC+8 ──▶ 觸發美股數據更新（前一交易日收盤）
   │                    POST /api/data/sync?market=us
   │
   └─ 15:00 UTC+8 ──▶ 觸發台股數據更新（當日收盤後）
                        POST /api/data/sync?market=tw

手動更新按鈕
   └─ 使用者點擊 ──▶ POST /api/data/sync?market={current}
                      前端顯示更新進度條
                      完成後 WebSocket 推送更新通知
```
